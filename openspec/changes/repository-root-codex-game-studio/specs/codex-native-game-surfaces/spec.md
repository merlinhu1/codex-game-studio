## ADDED Requirements

### Requirement: Custom agents use Codex TOML format
The system SHALL generate project-scoped Codex custom agents as `.codex/agents/*.toml` files.

#### Scenario: Agent TOML includes required fields
- **WHEN** a role agent is generated
- **THEN** its TOML file contains `name`, `description`, and `developer_instructions`

#### Scenario: Agent Markdown is not generated
- **WHEN** project agent surfaces are generated
- **THEN** no `.codex/agents/*.md` files are produced

### Requirement: Agent generation respects active engine
The system SHALL materialize only role agents relevant to the active engine and project mode.

#### Scenario: Godot project omits other engine specialists
- **WHEN** a Godot root project is initialized
- **THEN** `.codex/agents/godot-specialist.toml` exists
- **AND** `.codex/agents/unity-specialist.toml` and `.codex/agents/unreal-specialist.toml` do not exist

#### Scenario: Wrong-engine agent fails validation
- **WHEN** a wrong-engine custom agent file is present in `.codex/agents/`
- **THEN** project validation fails with a diagnostic naming the invalid file

### Requirement: Skills use repository skill package format
The system SHALL generate workflow, onboarding, and standards skills under `.agents/skills/<skill-name>/SKILL.md`.

#### Scenario: First-pass skills are generated
- **WHEN** a root project is initialized
- **THEN** `.agents/skills/cgs-start/SKILL.md`, `.agents/skills/cgs-setup-engine/SKILL.md`, `.agents/skills/cgs-bugfix/SKILL.md`, `.agents/skills/cgs-vertical-slice/SKILL.md`, `.agents/skills/cgs-standards-gameplay/SKILL.md`, and `.agents/skills/cgs-standards-tests/SKILL.md` exist

#### Scenario: Skill metadata is valid
- **WHEN** validation reads a generated `SKILL.md`
- **THEN** it finds `name` and `description` frontmatter

### Requirement: Runtime prompt mirrors remain explicit
The system SHALL keep `.codex/prompts/**` and `.codex/workflows/**` as CGS runtime artifacts until direct custom-agent orchestration is stable.

#### Scenario: Dry-run reports both surfaces
- **WHEN** `run --dry-run` prepares a role task
- **THEN** output includes the runtime prompt path under `.codex/prompts/**`
- **AND** the corresponding custom-agent path under `.codex/agents/*.toml`

### Requirement: Generated surfaces are freshness-validated
The system SHALL validate generated agent, skill, prompt, and workflow surfaces using deterministic provenance metadata.

#### Scenario: Stale generated surface fails validation
- **WHEN** a generated surface source hash does not match current source inputs
- **THEN** validation fails and names the stale surface

#### Scenario: Missing generated surface fails validation
- **WHEN** a required first-pass agent or skill surface is missing
- **THEN** validation fails and names the missing surface

### Requirement: Rules and hooks are not generated by default
The system SHALL NOT generate `.codex/rules/*.rules` for coding standards or `.codex/hooks.json` by default.

#### Scenario: Coding standards do not create permission rules
- **WHEN** standards skills are generated
- **THEN** no `.codex/rules/*.rules` file is created for those standards

#### Scenario: Hooks are absent by default
- **WHEN** a root project is initialized without an explicit future hooks feature
- **THEN** `.codex/hooks.json` does not exist
